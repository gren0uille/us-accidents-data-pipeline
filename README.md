# Аналитический конвейер обработки данных о ДТП в США

## О проекте
Проектирование и реализация аналитического пайплайна для обработки больших данных
о дорожно-транспортных происшествиях в США (2016–2023) с использованием
MPP-архитектуры и BI-визуализации.

**Датасет:** US Accidents — **7 728 394 записи**, 46 атрибутов, ~2.9 ГБ

## Задачи
- Спроектировать хранилище данных по схеме «звезда»
- Реализовать ETL-пайплайн: загрузка, очистка, трансформация
- Сформировать таблицы фактов и измерений в MPP-СУБД
- Подключить Apache Superset и построить аналитический дашборд

## Архитектура решения

**Схема «звезда»** — 4 таблицы:

| Таблица | Роль | Строк | Хранение |
|---------|------|-------|----------|
| `stage_accidents` | Staging-слой | 7 728 394 | heap, RANDOMLY |
| `dim_location` | Измерение (география) | 20 323 | heap, REPLICATED |
| `dim_weather` | Измерение (погода) | 1 681 | heap, REPLICATED |
| `fact_accident` | Таблица фактов | 7 728 394 | ao_column + zstd, BY(accident_id) |

**Пайплайн:**
```
CSV → stage_accidents → dim_location + dim_weather → fact_accident → ANALYZE → Superset
```

## Ключевые проектные решения
- **ao_column + компрессия zstd уровня 3** для таблицы фактов — оптимально для
  аналитических нагрузок (данные загружаются один раз, читаются многократно)
- **REPLICATED для измерений** — полная копия на каждом сегменте, избегаем
  Redistribute Motion при JOIN
- **Партиционирование по дате** (96 партиций, шаг 1 месяц) — ускорение запросов
  с фильтрацией по периоду
- **Сравнительный эксперимент** ключей распределения: distribution BY(accident_id)
  показал время выполнения **5 500 мс** против 10 499 мс при BY(severity)

## Ключевые аналитические результаты
- Калифорния лидирует по числу ДТП — **1 741 422** (×2 от Флориды на 2-м месте)
- Рост числа ДТП с **410К (2016) до 1.76М (2022)** — увеличение в 4.3 раза за 6 лет
- **79.7%** всех ДТП относятся к умеренной серьёзности (уровень 2)
- При погоде **Clear** средняя серьёзность выше (2.37), чем при Fair (2.13) —
  водители превышают скорость при хорошей видимости

## Стек
Arenadata DB (Greenplum), Apache Superset 6.0, Docker Compose, SQL
